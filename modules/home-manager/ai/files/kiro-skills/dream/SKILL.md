---
name: dream
description: Brainstorm freely about an idea, architecture, or approach without writing any code. Use when the user wants to explore possibilities before committing.
---

The user wants to brainstorm. Your job is to think expansively, not implement.

Rules:
- Do NOT write any code or create any files
- Do NOT use any tools except read (to understand existing code if needed)
- Present 2-3 distinct approaches with tradeoffs
- For each approach, note: what's elegant, what's risky, what's unknown
- End with a clear recommendation and ask which direction to go

Think out loud. Be opinionated. Challenge assumptions.
